# 日志收集与备份策略

## 日志规范

所有应用日志统一写入以下路径：

```
/data/logs/{项目名}/{服务名}/{日期}.log
```

示例：`/data/logs/aiops/web/2026-06-13.log`

日志格式要求：
- 每行一条日志记录
- 等级标记：[INFO]、[WARN]、[ERROR]、[FATAL]
- 包含时间戳、请求 ID、调用链信息
- 异常日志必须包含完整堆栈（最多 50 行，防止单条日志过大）

## 日志保留策略

| 日志类型 | 保留时间 | 存储位置 | 归档方式 |
|---------|---------|---------|---------|
| 应用日志 | 30 天 | 本地磁盘 | 按天轮转，压缩后归档到 MinIO |
| 访问日志 | 90 天 | 本地磁盘 | 按天轮转，压缩后归档到 ELK |
| 系统日志 | 90 天 | journald | 自动轮转 |
| 审计日志 | 365 天 | ELK | 冷热分离 |

## 日志采集架构

```
应用 Pod → 本地文件 → Filebeat → Kafka → Logstash → Elasticsearch → Kibana
```

注意：Filebeat 直接发送到 Elasticsearch 仅限开发环境。生产环境必须经过 Kafka。

## 备份策略

| 数据类型 | 备份频率 | 保留周期 | 存储位置 |
|---------|---------|---------|---------|
| 数据库全量 | 每日 02:00 | 7 天 | MinIO + 异地 |
| 数据库增量 | 每 30 分钟 | 7 天 | MinIO |
| 配置文件 | 变更后自动 | 90 天 | Git + MinIO |
| 容器镜像 | 每次构建 | 30 天 | Harbor |

## 备份恢复验证

- 数据库备份：每周一自动恢复到测试环境验证完整性
- 配置文件备份：每月抽取 10% 随机验证
- 异地备份：每月执行一次异地恢复演练
- 任何备份恢复失败必须在 24 小时内完成修复并重新备份
