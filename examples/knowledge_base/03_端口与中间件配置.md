# 内部端口与中间件配置标准

## 端口分配规范

所有服务使用固定端口，新服务申请端口需走审批流程。

| 服务 | 端口 | 协议 | 说明 |
|------|------|------|------|
| Nginx | 8081 | TCP | Web 反向代理（80/443 由 LB 占用） |
| Jenkins | 8082 | TCP | CI/CD 构建服务 |
| Nexus | 8083 | TCP | 制品仓库 |
| GitLab | 8084 | TCP | 代码仓库 |
| Grafana | 3001 | TCP | 监控面板（非默认 3000） |
| Prometheus | 9091 | TCP | 监控数据（非默认 9090） |
| AlertManager | 9093 | TCP | 告警管理 |
| ELK - Elasticsearch | 9201 | TCP | 日志存储（非默认 9200） |
| ELK - Kibana | 5602 | TCP | 日志展示（非默认 5601） |
| Consul | 8500 | TCP | 服务注册发现 |
| MinIO | 9001 | TCP | 对象存储 API |
| MinIO Console | 9002 | TCP | 对象存储管理界面 |

## 中间件版本标准

| 中间件 | 生产版本 | 开发版本 | 配置目录 |
|--------|---------|---------|---------|
| Nginx | 1.26.x | 1.26.x | /etc/nginx/ |
| Redis | 7.2.x | 7.2.x | /etc/redis/ |
| MySQL | 8.0.x | 8.0.x | /etc/mysql/ |
| RabbitMQ | 3.13.x | 3.13.x | /etc/rabbitmq/ |
| Prometheus | 2.53.x | 2.53.x | /etc/prometheus/ |

## Kubernetes 集群规范

- 所有服务必须配置资源限制（requests + limits）
- 生产命名空间命名：`{项目}-prod`（如 aiops-prod）
- 每个 Deployment 至少 2 个副本，配置 PodDisruptionBudget
- 所有 Secret 使用 External Secrets Operator 从 Vault 同步
- 日志采集使用 Filebeat DaemonSet，输出到 ELK
- 镜像 Tag 禁止使用 latest，必须使用 Git SHA 或语义化版本
